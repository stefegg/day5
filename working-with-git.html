Working with Git

1. Add files to your project

This can be done via the touch command or through the interface as usual

Example: $ touch index.html && touch main.css && touch main.js

2. Add new or modified files to staging

Once we add new files or modify existing files, we an tell git to track them by adding them to staging for the next commit (think of commits as checkpoints). This will let git know what files the next commit will consist of.

$ git add < filename >

OR

$ git add .

Your files are now on stage to be included in the next commit/checkpoint.

3. Add a Commit and a Commit message

With your changes staged, you can create the checkpoint in your work with the following command:

$ git commit -m "commit message"

This command creates the checkpoint for git! Pretty simple.

4. Push your code!

Finally, push your changes up to Github with the following command:

$ git push

You should see the following output (or similar):

    
  Counting objects: 4, done.
  Delta compression using up to 8 threads.
  Compressing objects: 100% (4/4), done.
  Writing objects: 100% (4/4), 548 bytes | 548.00 KiB/s, done.
  Total 4 (delta 0), reused 0 (delta 0)
  To https://github.com/stefegg/day5.git

If you get an error, you may have not initialized your git repository with $ git init
Or you may have initialized, but you have not yet modified any files since then using:

$ git add < file name > and $ git commit -m "commit message"
